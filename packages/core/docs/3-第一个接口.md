`befly` 采用的是 `约定大于配置` 的思路，所有项目相关的接口都要按照约定放在 `apis` 目录下。

接下来我们实现一个简单的 `复读机` 接口 `apis/echo.ts`，文件结构如下：

```bash
befly-demo/
├── apis
│   ├── echo.ts     # 复读机接口
├── node_modules
├── bun.lock
├── index.ts
├── package.json
└── tsconfig.json
```

接口文件的写法如下：

```typescript
import { Yes } from "befly";

export default {
    name: "复读机接口",
    method: "get",
    auth: false,
    fields: {
        words: {
            name: "文字内容",
            type: "string",
            min: 0,
            max: 999
        }
    },
    handler: async (befly, ctx) => {
        return Yes(ctx.body.words, {
            timestamp: Date.now(),
            message: "欢迎使用 Befly 框架"
        });
    }
};
```
